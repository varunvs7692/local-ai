# Security Policy

## Supported Versions

Currently, security updates are applied to the latest release of the NAP Multi-Agent Workspace. 

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly by contacting the maintainer via email at [varunvs7692@gmail.com](mailto:varunvs7692@gmail.com).

Please include the following details in your report:
- **Description**: A detailed description of the vulnerability.
- **Impact**: The potential impact or what an attacker could achieve.
- **Reproduction Steps**: Step-by-step instructions or a proof-of-concept (PoC) to reproduce the issue.
- **Environment**: Details of the environment (OS, node.js version, browser version, Ollama version, etc.) where the vulnerability was found.

We request that you do not publish the vulnerability or report it publicly (e.g., via GitHub Issues) until it has been investigated and resolved.

## Response Process

Upon receiving a vulnerability report:
1. We will acknowledge receipt of the report within **48 hours**.
2. We will investigate the issue and determine the appropriate fix.
3. Once a fix is prepared, we will publish a patch release and notify you.
4. We aim to resolve critical vulnerabilities within **7 days** of reporting.
